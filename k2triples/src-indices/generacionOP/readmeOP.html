Construcción de índices OP

sandra.alvarez@udc.es

En este directorio se incluye el script necesario para construír el índice OP codificado con Códigos de acceso directo(DAC). El comportamiento y por tanto también esta documentación, es idéntico a la generación de índices SP.

Formato del dataset

Para ejecutar este código es necesario tener el dataset RDF el mismo formato requerido en el proceso de construcción de los árboles de K2-triples:

  p1 s1 o1
  p2 s2 o2
  ...

Los tres elementos deben ser identificadores numéricos comenzando en el 1. Este fichero debe contener en binario cada triple utilizando tres enteros (de 4 bytes) en el siguiente orden predicado,sujeto,objeto.

Es imprescindible que la numeración siga las reglas de numeración especificadas en Compressed k2-Triples for Full-In-Memory RDF Engines en cuanto a las regiones S, O, SO, P. De lo contrario, pueden obtenerse falsos positivos en las consultas realizadas. Estas reglas de numeración se pueden resumir en:

  • Los elementos que participan en tripletas del dataset a veces como sujetos y a veces como objetos, estarán numerados consecutivamente de 1 ... |SO|
  • Los elementos que sólo actúan como sujetos irán numerados consecutivamente desde |SO|+1 .. |SO|+|S|
  • Los elementos que sólo actúan como objetos irán numerados consecutivamente desde |SO|+1 .. |SO|+|O|
  • Los elementos que actúan como predicados irán numerados desde 1...|P|

Además, para obtener una mayor compresibilidad de los datos es recomendable que dentro de cada uno de estos grupos, la asignación de identificadores siga el orden alfabético, ya que sujetos con el mismo prefijo se espera que estén descritos por predicados y objetos comunes.

Compilación

En la raíz se incluye un Makefile principal con el que se compilan todas las herramientas necesarias para ejecutar el script de creación del índice. Por lo tanto, sólo es necesario hacer:

  make clean all

Ejecución

Para iniciar el proceso de creación del índice OP sólo es necesario ejecutar el script principal:

./creacionIndice.sh -dataset.triples-

Tras la finalización del proceso, en el directorio raíz aparecen los ficheros que componen el índice OP comprimido utilizando DAC:

opDACC.ind
opDACC.dacc
opDACC.info
opDACC.dic

Herramientas incluídas en el proceso de construcción

El proceso completo de creación de índice está compuesto por tres secciones que pueden ser ejecutadas independientemente.

Construcción del índice DACC

Esta es la sección imprescindible para crear el índice. Puede invocarse ejecutando el script:

./creacionDACC.sh -dataset.triples-

Comprobación de corrección

Esta sección opcional comprueba que el índice se ha creado correctamente mediante la reconstrucción de la información original a partir de él. Se puede invocar mediante el siguiente script:

./comprobacionDACC.sh -dataset.triples-

Este script tendrá como salida las diferencias obtenidas en la reconstrucción, de modo que para un índice creado correctamente la salida esperada es:

DIFERENCIAS:
------------

Borrado de ficheros auxiliares

Elimina los archivos auxiliares creados durante el proceso de creación y comprobación. 

./borradoAUXDACC.sh
